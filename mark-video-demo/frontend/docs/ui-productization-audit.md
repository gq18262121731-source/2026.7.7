# 前端产品化 UI 审计

## 当前结论

前端已经具备 React + Vite + Tailwind + lucide 的基础，适合按“设计系统 + 组件化重构 + 截图验收”的路线推进。当前不需要接入 Figma MCP，也不需要引入新的大型 UI 库。

## 信息架构

- 主导航已经覆盖工作台、协同巡检、检测、记录、预警、风险、知识助手和系统状态。
- 协同巡检页是答辩主线，已经按田块建档、UAV dry-run、手机复查、报告闭环拆分。
- 需要保持“工作台型页面”结构：顶部状态、阶段进度、主操作区、右侧上下文。

## 设计系统问题

- 公共组件已有雏形：`PagePanel`、`StatusPill`、`MetricCard`、`EmptyState`、`ErrorNotice`。
- 视觉 token 还分散在全局 CSS 和页面 class 中，卡片、按钮、状态色、表格和空态应继续收敛。
- 风险与状态需要固定语义：normal、mild、moderate、severe、experimental、dry-run、smoke。
- 现有深色界面偏绿/青，应加入琥珀和红色风险语义，避免所有状态都像同一种高亮。

## 代码落地边界

- 不修改后端 API。
- 不改变业务状态流。
- 不删除 smoke、experimental、dry-run、fallback 等安全口径。
- 优先重构协同巡检页，再扩展到 Dashboard、记录、告警、风险预测和知识助手。

## 验收标准

- `npm run build` 必须通过。
- 页面不出现中文乱码。
- 协同巡检页应能清楚表达阶段、当前上下文、下一步操作和安全边界。
- 公共组件应能被后续页面复用，不把每个页面写成独立样式堆叠。
- 截图验收至少覆盖工作台首页和协同巡检页。
