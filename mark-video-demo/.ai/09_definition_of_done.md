# Definition of Done

一个功能不能因为“页面能显示”就算完成。

## 页面完成标准

页面必须满足：

- 可以正常访问
- 使用统一 Layout
- 使用统一主题
- 数据来自 API
- Loading 状态完整
- Error 状态完整
- Empty 状态完整
- 操作后有反馈
- 主要按钮可点击且行为明确
- 响应式不崩坏
- 适合录屏

## API 完成标准

API 必须满足：

- 路由存在
- 请求结构明确
- 响应结构明确
- 错误结构统一
- Mock 数据集中管理
- 前端可以成功调用
- 后续可替换真实 service

## 前端完成标准

前端功能必须满足：

- TypeScript 无类型错误
- ESLint 无明显错误
- 组件职责清晰
- 不使用未批准的 `any`
- 不在页面中写死业务 Mock
- 不破坏已有路由
- 不破坏已有样式系统

## 交互完成标准

交互必须满足：

- 点击有反馈
- 异步有 Loading
- 失败有 Error
- 无数据有 Empty
- 成功有 Toast 或可见结果
- 禁用状态明确

## 录屏完成标准

用于视频演示的功能必须满足：

- 首屏信息清晰
- 动画稳定
- 数据可预测
- 不依赖外网
- 不依赖真实模型
- 不出现明显报错
- 不出现空白页
- 不出现未完成占位文案

只有以上条件满足，才算完成。
