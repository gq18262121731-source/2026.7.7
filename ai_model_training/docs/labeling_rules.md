# 标注规范

## 标注格式

采用 YOLO 检测框格式。每张图片对应一个同名 `.txt` 文件，每行一个目标：

```text
class_id x_center y_center width height
```

坐标均为归一化坐标，范围为 0 到 1。

示例：

```text
0 0.5123 0.4312 0.1200 0.0800
1 0.2231 0.6120 0.0900 0.0600
```

## 手机近距离 RGB 图片

- 框住病斑主体区域，不要默认框整张叶片。
- 虫害目标优先框住虫体本身；虫体不可见但受害症状明显时，框受害区域。
- 多个分散病斑尽量分别标注。
- 病斑密集连片时，可标为一个连续区域。
- 不确定类别先放入 `uncertain/` 待复核，不强行归类。
- 模糊、弱光、遮挡但仍可识别的图片可保留，建议进入鲁棒性测试集或单独标注来源。

## 无人机远距离或多光谱视角

- 单个病斑不可见时，标注疑似病害区域或异常冠层区域。
- 虫体不可见但存在明显受害区域时，标注受害区域，不假装看见虫体。
- 正射图或超大图应先切片再标注，或标注后同步转换到切片坐标。
- 小目标过小且无法确认时，不强行标注，避免引入训练噪声。
- 多光谱伪 RGB 或指数图必须记录生成方式，确保训练和推理一致。

## 质量审核

每批标注至少完成一次复核，检查：

| 检查项 | 要求 |
| --- | --- |
| 类别正确性 | 类别与专家判断一致 |
| 框贴合度 | 框尽量贴合目标主体 |
| 漏标 | 明显目标不得遗漏 |
| 误标 | 正常区域不得误标为病虫害 |
| 重复框 | 同一目标不得重复标注 |
| 尺度一致性 | 同类目标标注尺度保持一致 |
| 正常图 | 正常图片应为空标签文件或无标签文件，按数据加载策略统一 |
| 不确定样本 | 单独隔离，不进入训练集 |

建议流程：一人标注 -> 一人复核 -> 争议样本交由农技老师或专业人员确认 -> 不确定样本进入 `uncertain/`。
