# 前端产品化 UI 审查与优化第二轮报告

日期：2026-07-05

## 1. 本轮目标

从真实用户视角检查 `mark-video-demo` 前端页面结构，将页面从“演示项目风格”进一步调整为更接近市面上业务软件的农业巡检系统。

本轮重点优化：

- 图像检测页面。
- 历史记录页面。
- 系统配置页面。
- 全局组件一致性延续。

## 2. 主要问题

优化前主要问题：

- 检测页信息分布偏演示，操作焦点不够明确。
- 历史页缺少记录总览和选中态，扫描效率不高。
- 系统管理页像资料展示，不像配置状态中心。
- 部分按钮和卡片仍有临时演示感。

## 3. 已完成优化

### 图像检测

文件：

- `mark-video-demo/frontend/src/pages/DetectPage.tsx`

改动：

- 改为三栏操作台布局：输入选择、图像检测框、分析结论。
- 样本选择更紧凑。
- 分析流程状态更清晰。
- 移除“保存诊断报告”假按钮，避免误导用户。
- 结论区明确展示首要结果、风险、置信度、建议和边界。

### 历史记录

文件：

- `mark-video-demo/frontend/src/pages/HistoryPage.tsx`

改动：

- 增加记录总数、高风险、已完成三个指标。
- 表格行增加选中态。
- 详情区分层展示图像、分析结论和关键字段。
- 增加加载、空状态和错误提示。

### 系统配置

文件：

- `mark-video-demo/frontend/src/pages/SettingsPage.tsx`

改动：

- 改为配置状态页。
- 增加模型数量、近景样本、无人机样本、安全口径指标。
- 模型路线、数据源、诊断与安全边界分区更清楚。
- 页面显式保留辅助诊断安全口径。
